* goal
  * supported 
    * access control models & 
    * patterns

* SIMPLEST model
  * **subject** (user OR service) requests access
    * -- to an -- **object** (resource OR entity)
    * -- to -- perform an **action** (_read_, _write_, or _delete_)

| Model                                                                            | Description                                                                                                                                        | Example                                                                                                                                                                                                  |
|----------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| ACL                                                                              | [here](https://en.wikipedia.org/wiki/Access_control_list) <br/> MOST basic & simple                                                                | [model](https://github.com/casbin/casbin/blob/master/examples/basic_model.conf) / [policy](https://github.com/casbin/casbin/blob/master/examples/basic_policy.csv)                                       |
| ACL + [superuser](https://en.wikipedia.org/wiki/Superuser)                       |                                                                                                                                                    | [model](https://github.com/casbin/casbin/blob/master/examples/basic_with_root_model.conf) / [policy](https://github.com/casbin/casbin/blob/master/examples/basic_policy.csv)                             |
| ACL WITHOUT users                                                                | _Example:_ API-only OR device access                                                                                                               | [model](https://github.com/casbin/casbin/blob/master/examples/basic_without_users_model.conf) / [policy](https://github.com/casbin/casbin/blob/master/examples/basic_without_users_policy.csv)           |
| ACL WITHOUT resources                                                            | permissions apply \| <br/> &nbsp;&nbsp; resource types <br/> &nbsp;&nbsp; NOT \| specific instances (_Example:_ "write-article", "read-log")       | [model](https://github.com/casbin/casbin/blob/master/examples/basic_without_resources_model.conf) / [policy](https://github.com/casbin/casbin/blob/master/examples/basic_without_resources_policy.csv)   |
| RBAC                                                                             | [here](https://en.wikipedia.org/wiki/Role-based_access_control)                                                                                    | [model](https://github.com/casbin/casbin/blob/master/examples/rbac_model.conf) / [policy](https://github.com/casbin/casbin/blob/master/examples/rbac_policy.csv)                                         |
| RBAC + resource roles                                                            | users & resources can have <br/> &nbsp;&nbsp; roles OR <br/> &nbsp;&nbsp; group memberships                                                        | [model](https://github.com/casbin/casbin/blob/master/examples/rbac_with_resource_roles_model.conf) / [policy](https://github.com/casbin/casbin/blob/master/examples/rbac_with_resource_roles_policy.csv) |
| RBAC + domains/tenants                                                           | users can have DIFFERENT roles \| <br/> &nbsp;&nbsp; DIFFERENT domains OR <br/> &nbsp;&nbsp; DIFFERENT tenants                                     | [model](https://github.com/casbin/casbin/blob/master/examples/rbac_with_domains_model.conf) / [policy](https://github.com/casbin/casbin/blob/master/examples/rbac_with_domains_policy.csv)               |
| ReBAC                                                                            | Relationship-Based Access Control                                                                                                                  | [model](https://github.com/casbin/casbin/blob/master/examples/rebac_model.conf) / [policy](https://github.com/casbin/casbin/blob/master/examples/rebac_policy.csv)                                       |
| [ABAC](https://en.wikipedia.org/wiki/Attribute-Based_Access_Control)             | uses attributes \| rules <br/> _Example:_  `resource.Owner`                                                                                        | [model](https://github.com/casbin/casbin/blob/master/examples/abac_model.conf)                                                                                                                           |
| [PBAC](/docs/PBAC.md)                                                            | == Authorization is driven -- by -- rule-based policies <br/> uses: dynamic decisions & context-aware decisions                                    |                                                                                                                                                                                                          |
| [BLP (Bell–LaPadula)](https://en.wikipedia.org/wiki/Bell%E2%80%93LaPadula_model) | Formal model + security labels & clearances                                                                                                        | [model](https://github.com/casbin/casbin/blob/master/examples/blp_model.conf)                                                                                                                            |
| [Biba](https://en.wikipedia.org/wiki/Biba_Model)                                 | restricts information flow <br/> &nbsp;&nbsp; Reason:🧠prevent unauthorized changes🧠                                                              | [model](https://github.com/casbin/casbin/blob/master/examples/biba_model.conf)                                                                                                                           |
| [LBAC](/docs/LBAC.md)                                                            | == \| lattice-based framework, confidentiality + integrity                                                                                         | [model](https://github.com/casbin/casbin/blob/master/examples/lbac_model.conf)                                                                                                                           |
| [OrBAC](/docs/OrBAC.md)                                                          | == RBAC extended / abstraction layers -- for -- MULTI-organization policies                                                                        | [model](https://github.com/casbin/casbin/blob/master/examples/orbac_model.conf) / [policy](https://github.com/casbin/casbin/blob/master/examples/orbac_policy.csv)                                       |
| [UCON](/docs/UCON.mdx)                                                           | enable <br/> &nbsp;&nbsp; ongoing authorization <br/> &nbsp;&nbsp; mutable attributes <br/> &nbsp;&nbsp; obligations <br/> &nbsp;&nbsp; conditions |                                                                                                                                                                                                          |
| IP Match                                                                         | match -- by -- IP address OR CIDR for network-level control                                                                                        | [model](https://github.com/casbin/casbin/blob/master/examples/ipmatch_model.conf) / [policy](https://github.com/casbin/casbin/blob/master/examples/ipmatch_policy.csv)                                   |
| RESTful                                                                          | Path patterns (ej: `/res/*`, `/res/:id`) + HTTP methods                                                                                            | [model](https://github.com/casbin/casbin/blob/master/examples/keymatch_model.conf) / [policy](https://github.com/casbin/casbin/blob/master/examples/keymatch_policy.csv)                                 |
| Deny-override                                                                    | if you have allow policy + deny policies -> ALWAYS deny policy overrides                                                                           | [model](https://github.com/casbin/casbin/blob/master/examples/rbac_with_not_deny_model.conf) / [policy](https://github.com/casbin/casbin/blob/master/examples/rbac_with_deny_policy.csv)                 |
| Allow-and-deny                                                                   | requires explicit allow AND no deny; if both match, deny wins                                                                                      | [model](https://github.com/casbin/casbin/blob/master/examples/rbac_with_deny_model.conf) / [policy](https://github.com/casbin/casbin/blob/master/examples/rbac_with_deny_policy.csv)                     |
| Priority                                                                         | == order the policy rules / first match wins                                                                                                       | [model](https://github.com/casbin/casbin/blob/master/examples/priority_model.conf) / [policy](https://github.com/casbin/casbin/blob/master/examples/priority_policy.csv)                                 |
| Explicit Priority                                                                | priority as explicit numeric field per policy rule                                                                                                 | [model](https://github.com/casbin/casbin/blob/master/examples/priority_model_explicit.conf) / [policy](https://github.com/casbin/casbin/blob/master/examples/priority_policy_explicit.csv)               |
| Subject-Priority                                                                 | priority based on role hierarchy; more specific role wins                                                                                          | [model](https://github.com/casbin/casbin/blob/master/examples/subject_priority_model.conf) / [policy](https://github.com/casbin/casbin/blob/master/examples/subject_priority_policy.csv)                 |
